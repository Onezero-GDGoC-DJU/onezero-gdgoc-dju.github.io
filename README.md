# GDGoC 동아리 홈페이지 프로젝트

이 저장소는 비정기 스터디 형태로 진행되는 동아리 홈페이지 제작 프로젝트입니다.
이 프로젝트는 GitHub Pages 기능을 이용해 배포되는 정적 웹사이트 프로젝트입니다.

`codex cli` 또는 `gemini cli`를 활용한 바이브 코딩을 중심으로, 실제 결과물을 함께 만들면서 프론트엔드와 협업 흐름을 익히는 것을 목표로 합니다.

## 프로젝트 목표

- 동아리 홈페이지 완성
- 프론트엔드 기초 지식 습득
- GitHub / Git 사용에 익숙해지기

## 진행 방식

- 비정기적으로 모여 작업합니다.
- `codex cli` 또는 `gemini cli`를 활용해 아이디어를 빠르게 구현합니다.
- GitHub Pages에 반영되는 결과물을 기준으로 화면과 구조를 개선합니다.
- 직접 코드를 수정하고, 변경 사항을 리뷰하며 결과물을 발전시킵니다.

## 커뮤니케이션

- 주 소통 채널은 Discord입니다.
- 이슈 또는 Pull Request가 생성되면 Discord에 봇 알림이 전송됩니다.
- 진행 상황 공유, 작업 논의, 리뷰 요청도 Discord를 중심으로 이루어집니다.

## 작업 규칙

- 운영진이 작업할 이슈를 미리 생성합니다.
- 참여자는 원하는 이슈를 선택한 뒤 브랜치를 만들어 작업합니다.
- 작업이 끝나면 Pull Request를 생성합니다.
- 운영진이 내용을 확인한 뒤 `main` 브랜치에 머지합니다.

기본 작업 흐름은 아래와 같습니다.

1. GitHub Issue 확인
2. 작업할 이슈 선택
3. 브랜치 생성 후 작업 진행
4. Pull Request 생성
5. 운영진 리뷰 후 머지

협업 시에는 아래 규칙을 따릅니다.

- `main` 브랜치에 직접 push하지 않습니다.
- 한 개의 이슈는 가능한 한 한 가지 작업만 다루도록 합니다.
- 작업을 시작할 때는 해당 이슈에 표시하거나 Discord에서 먼저 공유합니다.
- Pull Request에는 어떤 작업을 했는지 짧고 명확하게 작성합니다.
- 가능하면 PR 본문에 관련 이슈 번호를 함께 적습니다.

## 참여 방법

함께 하고 싶은 사람은 운영진에게 연락해 참여할 수 있습니다.

프로젝트는 결과물 완성뿐 아니라, 협업 경험과 개발 도구 적응까지 포함한 실습형 스터디를 지향합니다.
